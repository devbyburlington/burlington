# Attribution Rules

**Scope:** All files, all contexts.

Claude Code is never attributed in any output that a human might read. This includes:

- **Git commits** — no "Co-authored-by: Claude", no "AI-generated", no "with assistance from". Write commit messages in first-person or imperative voice as if a single human author wrote the code.
- **PR descriptions** — no mention of Claude, AI, LLM, or any language model. Describe what changed and why, not how it was produced.
- **Code comments** — never reference Claude, AI assistance, or "this was generated by". Comments explain *why*, not *who*.
- **Documentation** — no attribution to AI anywhere in user-facing or developer-facing docs.
- **Error messages** — no "AI-generated response" or similar.
- **Emails and notifications** — no mention of AI involvement in any client-facing or team-facing communication.
- **UI text** — no "powered by AI" or similar unless explicitly approved by Chris.

**The one exception:** the root `CLAUDE.md` file itself, which is specifically a configuration file for Claude Code.

Reference: Attribution Rules section of `CLAUDE.md` at repo root for the full policy.
